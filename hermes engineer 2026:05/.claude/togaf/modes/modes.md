# TOGAF 開發模式定義

## 輕量模式 (Lightweight)
適用：快速驗證、MVP、個人專案、時間緊迫

產出文件：
- Phase A（架構願景）— 必須
- Phase B（業務架構）— 簡化版
- Requirements.md — 只做 FR + NFR

跳過：Phase C, D, E, F, G, H

產出時間目標：30 分鐘內完成

---

## 標準模式 (Standard)
適用：一般商業專案、團隊開發、有明確需求

產出文件：
- Phase A（架構願景）— 完整
- Phase B（業務架構）— 完整
- Phase C（資訊系統架構）— 完整
- Phase D（技術架構）— 完整
- Phase E（機會與解決方案）— 完整
- Requirements.md — 完整

跳過：Phase F, G, H（上線後再補）

產出時間目標：1-2 小時

---

## 企業模式 (Enterprise)
適用：大型專案、多團隊、企業客戶、高合規要求

產出文件：全部 Phase A-H + Requirements.md

額外要求：
- 每個 Phase 需要利害關係人核准
- ADR 記錄所有重大決策
- 變更控制流程完整執行

產出時間目標：半天以上
